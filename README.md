# QREADY AI HEMS PRO (Docker Edition)

ECHONET Lite規格を利用して、太陽光発電システム（QREADY-R59-1等）からリアルタイムに電力を取得し、蓄電池導入のシミュレーションを行うWebダッシュボードです。Dockerコンテナ化により、24時間安定した監視環境を容易に構築できます。

## 🚀 主な機能

- **リアルタイム監視**: 発電量、消費電力、売買電状況を5秒ごとに更新。
- **発電予測統合**: Open-Meteo APIと連携し、設置地点の気候に基づいた発電予想を表示。
- **蓄電池シミュレーター**: 
  - **日次**: 夜間電力を自給するために必要な最小容量を算出。
  - **年次**: 月ごとの実績データから、季節ごとの推奨蓄電池サイズをリスト表示。
- **電気料金概算**: 九州電力「電化でナイト・セレクト21」等の単価に基づいたリアルタイム算出。

## 🛠 ネットワーク構成

- **構成**: 有線ルータ(無線) ↔ RE305(中継機) ↔ パワコン(QREADY-R59-1 有線接続)
- **環境**: Docker / Docker Compose
- **バックエンド**: Python 3.10-slim (Flask)
- **データベース**: SQLite3 (ホスト側ディレクトリへ永続化)

### 1. リポジトリの準備

```bash
git clone <your-repository-url>

2. 環境変数の設定

.env ファイルを作成し、設置環境に合わせた情報を設定してください。
Plaintext

HEMS_IP=192.168.0.xxx      # パワコンの固定IPアドレス

HEMS_LON=130.xxxx          # 設置場所の経度

3. コンテナの起動
Bash

docker-compose up -d

起動後、ブラウザで http://<実行マシンのIP>:8000 にアクセスしてください。
📂 ディレクトリ構成

    app.py: メインプログラム（Flask & Data Collector）

    Dockerfile: コンテナイメージ定義

    docker-compose.yml: コンテナ実行定義（ボリューム・環境変数）

    .env: 接続先・座標設定（Git非推奨）

    data/: データベース保存用ディレクトリ（自動作成）

⚠️ 注意事項

    コンテナを停止しても、data/energy.db に保存された実績データは保持されます。

    パワコンとコンテナ実行ホストが同一ネットワーク内にあることを確認してください。
